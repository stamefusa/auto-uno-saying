# UNO自動宣言装置

スマートフォンのインカメラで手札を認識し、残り1枚になったら自動でUNOを宣言するシステム。

---

## システム構成

```
web/        Webアプリ（カメラ認識・UNO演出・BLE通信）
firmware/   ESP32ファームウェア（LED + サーボ制御）※未実装
docs/       要件定義・タスク管理ドキュメント
```

---

## モード

### 通常モード
- 手札が1枚になったことを検知すると音声を再生
- 画面中央に「UNO」をシンプルに表示
- 画面タップで消去・次の検知へリセット

### スーパーUNOアピールモード
- 手札が1枚になったことを検知すると音声を再生
- 「UNO」が画面いっぱいにスタンプ演出で表示
- 背景がゆっくり色変化し、大量の「UNO」文字が右から左へ流れる（ニコニコ動画風）
- 画面タップで消去・次の検知へリセット
- ESP32へのBLE通知（実装予定）

---

## カード認識アルゴリズム

Canvas API のみで実装（外部ライブラリ不使用）。

1. 200ms ごとにガイド枠内の映像フレームを取得
2. 各ピクセルを RGB→HSV 変換し、UNO固有の色相範囲（赤・青・緑・黄）かつ高彩度のピクセルを抽出
3. 白マスクの積分画像を構築し、近傍に白ピクセルがあるUNO色ピクセルのみをカード領域とみなす
4. 面積比で `none / single / multiple` を判定
5. `multiple → single` の遷移後、3フレーム連続で `single` が続いたらUNO宣言

## 動作環境

| コンポーネント | 要件 |
|---|---|
| Webアプリ | Chrome for Android / HTTPS必須 |
| BLEデバイス | ESP32 / M5Atom Lite / AtomS3 Lite（スーパーモードのみ） |
| サーボ | SG90（PWM制御）（スーパーモードのみ） |
| 電源 | 単3×4 または USB 5V（サーボ用外部電源） |

---

## セットアップ

### Webアプリ（Vercel）

1. [vercel.com](https://vercel.com) でリポジトリをインポート
2. Root Directory: `web` を指定
3. スマートフォンの Chrome でデプロイ先 URL にアクセス
4. カメラ使用許可を承認してモードを選択

### ローカル開発

```bash
cd web
npm install
npm run dev
```

デバッグ画面（閾値スライダー付き）は `/debug` で確認できる。

### ESP32ファームウェア（未実装）

1. `firmware/` ディレクトリを Arduino IDE または PlatformIO で開く
2. ESP32 ボードへ書き込み
3. サーボ・LED を配線し、外部電源を接続（GND共通）

## 制約事項

- Chrome for Android 専用（iOS Safari 非対応）
- HTTPS 環境が必須（getUserMedia / Web Bluetooth API の要件）
- 照明条件・カードの重なり方により認識精度が変動する（`/debug` ページで閾値調整可能）
