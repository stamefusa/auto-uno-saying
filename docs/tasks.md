# UNO自動宣言装置 タスク一覧

要件定義書（SystemRequirements.md）をもとに分解したタスク。
上から順番に実施する。

---

## Phase 1: プロジェクト基盤

- [ ] **T01** リポジトリ構成の整備
  - ディレクトリ構成を決定（例: `web/`, `firmware/`, `docs/`）
  - README.md を作成（概要・セットアップ手順）

---

## Phase 2: Webアプリ — カメラ・UI基盤

- [ ] **T02** 静的Webアプリの雛形作成
  - HTML / CSS / JS の最小構成を用意
  - HTTPS配信を前提とした構成（GitHub Pages等）

- [ ] **T03** インカメラ映像の取得とプレビュー表示
  - `getUserMedia` でフロントカメラを指定して取得
  - `<video>` 要素にリアルタイムプレビューを表示
  - カメラ未許可時のエラーメッセージ表示

- [ ] **T04** 手札ガイド枠の表示
  - カメラプレビュー上に枠（オーバーレイ）を描画
  - 枠内を画像解析対象とする

---

## Phase 3: Webアプリ — 手札枚数認識

- [ ] **T05** `<canvas>` へのフレームキャプチャ実装
  - 一定間隔でビデオフレームをキャンバスへ描画
  - ガイド枠内の領域を切り出す

- [x] **T06** ~~OpenCV.js の導入~~ → **スキップ**（Canvas API のみで実装する方針に変更）

- [x] **T07** カード有無判定ロジック実装（UNO色相フィルタ + 白枠隣接チェック）
  - RGB → HSV 変換で UNO固有の色相範囲（赤・青・緑・黄）かつ高彩度のピクセルを抽出
  - 白マスク（低彩度・高輝度）の積分画像を構築し、近傍に白ピクセルがある UNO色ピクセルのみをカード領域とみなす
  - 面積比で `none / single / multiple` を判定（RATIO_NONE: 3%, RATIO_SINGLE: 9%）
  - 判定結果とデバッグオーバーレイ（カードマスク・白マスク・面積比バー）を画面に表示

---

## Phase 4: Webアプリ — UNO判定

- [ ] **T08** UNO判定ロジック実装
  - 直前の状態と現在の状態を比較
  - `multiple → single` への変化を検知してUNOと判定
  - 同一状態での連続発火を防止するフラグ管理

- [ ] **T09** UNO表示の実装
  - UNO判定時に画面中央へ大きく「UNO」を表示
  - 数秒間表示後に自動消去

---

## Phase 5: Webアプリ — BLE接続

- [ ] **T10** Web Bluetooth API によるBLEデバイス検索・接続
  - BLE接続ボタンを設置
  - `navigator.bluetooth.requestDevice` でデバイスを選択
  - サービスUUID / CharacteristicUUID の定数定義
  - 接続状態（接続中 / 接続済み / 未接続）を画面に表示

- [ ] **T11** BLE通知送信の実装
  - UNO判定時に Characteristic へ `0x01` または `"UNO"` を書き込む
  - BLE未接続時のエラー処理（送信スキップ＋通知）

---

## Phase 6: ESP32ファームウェア

- [ ] **T12** ESP32 BLE Peripheralのセットアップ
  - 固定デバイス名でアドバタイズ
  - サービスUUID / CharacteristicUUID を定義
  - 書き込み可能 Characteristic を1つ用意

- [ ] **T13** BLE受信ハンドラの実装
  - Characteristicへの書き込みを検知
  - 受信データを解析（`0x01` or `"UNO"` を判定）

- [ ] **T14** LED制御の実装
  - UNO通知受信時にLEDを点灯
  - 固定時間後に自動消灯

- [ ] **T15** サーボモーター制御の実装
  - PWMでSG90を制御
  - 受信時に 0° → 60° へ回転（旗展開）
  - 一定時間後に 60° → 0° へ戻す（旗収納）

- [ ] **T16** 電源・GND配線の確認
  - サーボを外部電源（単3×4 or USB 5V）へ接続
  - ESP32とサーボのGNDを共通化

---

## Phase 7: 結合テスト・受け入れ確認

- [ ] **T17** Webアプリ単体動作確認
  - インカメラ映像が表示されること
  - 手札枚数が概算で判定できること
  - 1枚になった際にUNO表示されること

- [ ] **T18** BLE通信の結合テスト
  - WebアプリからESP32へBLE通知が届くこと
  - ESP32がLEDを点灯すること
  - サーボがUNO旗を展開すること

- [ ] **T19** 受け入れ基準の全項目確認（SystemRequirements.md § 10）

## 備考

- Chrome for Android + HTTPS 環境が必須（iOS Safari は対象外）
- カードの認識精度は照明条件・重なり方に依存するため、試作段階では許容範囲を広めに設定する
