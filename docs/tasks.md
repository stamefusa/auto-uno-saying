## Phase 2: Webアプリ — カメラ・UI基盤

- [x] **T02** 静的Webアプリの雛形作成
  - React + Vite + TypeScript + Tailwind CSS v4 構成
  - Vercel デプロイ設定（vercel.json）

- [x] **T03** インカメラ映像の取得とプレビュー表示
  - `getUserMedia` でフロントカメラを指定して取得
  - `<video>` 要素にリアルタイムプレビューを表示
  - カメラ未許可時のエラーメッセージ表示

- [x] **T04** 手札ガイド枠の表示
  - カメラプレビュー上に枠（オーバーレイ）を描画
  - 枠内を画像解析対象とする

---

## Phase 3: Webアプリ — 手札枚数認識

- [x] **T05** `<canvas>` へのフレームキャプチャ実装
  - 一定間隔でビデオフレームをキャンバスへ描画
  - ガイド枠内の領域を切り出す

- [x] **T06** ~~OpenCV.js の導入~~ → **スキップ**（Canvas API のみで実装する方針に変更）

- [x] **T07** カード有無判定ロジック実装（UNO色相フィルタ + 白枠隣接チェック）
  - RGB → HSV 変換で UNO固有の色相範囲（赤・青・緑・黄）かつ高彩度のピクセルを抽出
  - 白マスク（低彩度・高輝度）の積分画像を構築し、近傍に白ピクセルがある UNO色ピクセルのみをカード領域とみなす
  - 面積比で `none / single / multiple` を判定（RATIO_NONE: 3%, RATIO_SINGLE: 9%）
  - 判定結果とデバッグオーバーレイ（カードマスク・白マスク・面積比バー）を画面に表示

## Phase 3.5: Webアプリ — ルーティングとデバッグ画面分離

- [ ] **T07b** ルーティング導入とページ分離
  - `react-router-dom` を導入
  - `/` → 本番アプリ（T08以降で実装するUNO判定画面）

## Phase 4: Webアプリ — UNO判定・演出

- [ ] **T08** UNO判定ロジック実装
  - 直前の状態と現在の状態を比較
  - `multiple → single` への変化を検知してUNOと判定
  - 同一状態での連続発火を防止するフラグ管理

- [ ] **T09** モード選択UIの実装
  - 初期画面またはメニューでモードを選択できるようにする
  - 選択肢: 通常モード / スーパーUNOアピールモード

- [ ] **T10** 音声ファイルの準備と再生実装
  - 通常モード用音声: 「UNO」と1回言う音声ファイルを用意（`public/audio/uno.mp3` など）
  - スーパーモード用音声: 「UNO」を複数回・激しく言う音声ファイルを用意
  - UNO判定時に Web Audio API または `<audio>` 要素で再生

- [ ] **T11** UNO表示演出の実装（通常モード）
  - UNO判定時に画面中央へ大きく「UNO」をシンプルに表示
  - 画面をタップしたら消去

- [ ] **T12** UNO表示演出の実装（スーパーUNOアピールモード）
  - UNO判定時に画面中央へ「UNO」を点滅・拡縮など激しく表示
  - 画面をタップしたら消去

## Phase 5: Webアプリ — BLE接続（スーパーUNOアピールモードのみ）

- [ ] **T13** Web Bluetooth API によるBLEデバイス検索・接続
  - BLE接続ボタンをスーパーモード時のみ表示
  - `navigator.bluetooth.requestDevice` でデバイスを選択
  - サービスUUID / CharacteristicUUID の定数定義
  - 接続状態（接続中 / 接続済み / 未接続）を画面に表示

- [ ] **T14** BLE通知送信の実装
  - UNO判定時に Characteristic へ `0x01` を書き込む
  - BLE未接続時はスキップ（エラーにしない）

---

## Phase 6: ESP32ファームウェア

- [ ] **T15** ESP32 BLE Peripheralのセットアップ
  - 固定デバイス名でアドバタイズ
  - サービスUUID / CharacteristicUUID を定義
  - 書き込み可能 Characteristic を1つ用意

- [ ] **T16** BLE受信ハンドラの実装
  - Characteristicへの書き込みを検知
  - 受信データを解析（`0x01` を判定）

- [ ] **T17** LED制御の実装
  - UNO通知受信時にLEDを点灯
  - 固定時間後に自動消灯

- [ ] **T18** サーボモーター制御の実装
  - PWMでSG90を制御
  - 受信時に 0° → 60° へ回転（旗展開）
  - 一定時間後に 60° → 0° へ戻す（旗収納）

- [ ] **T19** 電源・GND配線の確認
  - サーボを外部電源（単3×4 or USB 5V）へ接続
  - ESP32とサーボのGNDを共通化

## 備考

- Chrome for Android + HTTPS 環境が必須（iOS Safari は対象外）
- カードの認識精度は照明条件・重なり方に依存するため、試作段階では許容範囲を広めに設定する
- 閾値は実行環境（照明・カメラ特性）に依存するため `/debug` 画面でチューニングすること
- BLE連携はスーパーUNOアピールモード専用
