# UNO自動宣言装置 要件定義書（試作版）

## 1. 文書概要

### 1.1 目的
UNOプレイ中に、プレイヤーの手札が残り1枚になったことをスマートフォンのインカメラで検知し、Webアプリ上で「UNO」と表示するとともに音声を再生する。
スーパーUNOアピールモード選択時は、BLE接続された外部デバイスへも通知する。外部デバイスは通知を受けるとLEDを点灯し、サーボモーターを動作させて「UNO旗」を跳ね上げる。

### 1.2 対象範囲
本要件定義書は以下の2要素を対象とする。

- スマートフォン上で動作するWebアプリケーション
- BLE接続されるESP32 / M5Atom系デバイス（LED点灯およびサーボ動作）

### 1.3 試作範囲外
本試作では以下は対象外とする。

- UNOカードの種類（色・数字・アクションカード）の識別
- 対戦相手の手札監視
- UNO公式ルールとの厳密な一致
- iOS Safariなど全ブラウザ対応
- インターネット経由の遠隔通信
- プレイ履歴保存

---

# 2. システム概要

## 2.1 システム構成

    プレイヤー手札
          │
          ▼
    スマートフォン（インカメラ）
          │
          ▼
    Webアプリ（画像処理）
          │
          ├─ UNO表示・音声再生（全モード共通）
          │
          └─ BLE通信（スーパーUNOアピールモードのみ）
                │
                ▼
          ESP32 / M5Atom
                │
                ├─ LED点灯
                └─ サーボ動作 → UNO旗展開

## 2.2 動作モード

### 通常モード
1. プレイヤーがスマートフォンを首から下げ、インカメラで手札を映す
2. Webアプリが手札状態（none / single / multiple）をリアルタイム認識
3. 手札が「複数枚 → 1枚」に変化したことを検知
4. Web画面に「UNO」とシンプルに表示
5. 音声ファイル（「UNO」と1回言う）を再生
6. 画面をタップすると表示が消える

### スーパーUNOアピールモード
1. 通常モードと同様に手札変化を検知
2. Web画面に「UNO」を点滅・拡縮など激しく表示
3. 音声ファイル（「UNO」を複数回・激しく言う）を再生
4. 画面をタップすると表示が消える
5. BLEでESP32へ通知
6. ESP32がLEDを点灯
7. サーボモーターを動かし、UNO旗を跳ね上げる

# 3. Webアプリ要件

## 3.1 動作環境

### 3.1.1 対応端末
- スマートフォン

### 3.1.2 対応ブラウザ
- Google Chrome for Android

### 3.1.3 配信方式
- HTTPSで配信される静的Webアプリ

---

## 3.2 機能要件

### 3.2.1 カメラ取得
- インカメラの映像を取得できること
- カメラ使用許可をユーザーに要求すること
- カメラ映像を画面上にプレビュー表示すること

### 3.2.2 手札ガイド表示
- 手札を映す範囲を画面上に枠で表示する
- 画像解析はこの枠内を対象とする

### 3.2.3 手札状態認識
- カメラ映像から手札状態（none / single / multiple）を判定できること
- 判定方式: RGB→HSV変換 + UNO色相フィルタ（赤・青・緑・黄）+ 白枠隣接チェック（積分画像）
- UNOカードの種類識別は不要
- 画像処理は端末内で実行する

### 3.2.4 UNO判定
- 直前の状態と現在の状態を比較する
- `multiple → single` に変化した場合にUNOと判定する
- 同一状態での連続発火を防止する

### 3.2.5 モード選択
- 起動時または設定画面で通常モード / スーパーUNOアピールモードを選択できること

### 3.2.6 音声再生
- UNO判定時に音声ファイルを再生する
- 通常モード: 「UNO」と1回言う音声
- スーパーUNOアピールモード: 「UNO」を複数回・激しく言う音声

### 3.2.7 画面表示
- 現在の手札状態（none / single / multiple）を表示
- UNO発生時に画面中央へ「UNO」と表示
- 画面をタップすると表示が消える
- 通常モード: シンプルな表示
- スーパーUNOアピールモード: 点滅・拡縮など激しい演出

### 3.2.8 BLE接続（スーパーUNOアピールモードのみ）
- BLEデバイスを検索可能
- ユーザー操作で接続開始
- 接続状態を表示

### 3.2.9 BLE通知送信（スーパーUNOアピールモードのみ）
- UNO判定時にESP32へデータ送信
- 送信データ: `0x01`
- BLE未接続時はスキップ（エラーにしない）

### 3.2.10 エラー処理
- カメラ未許可時に警告表示
- BLE接続失敗時に通知
- 認識失敗時でもアプリが停止しない

### 3.2.11 デバッグ画面（/debug）
- `/debug` パスで開発・チューニング専用画面にアクセスできること
- デバッグオーバーレイ（カードマスク・白マスク・面積比バー）を常時表示
- 以下の閾値をスライダーでリアルタイム調整できること
  - SAT_THRESHOLD（彩度閾値）
  - RATIO_NONE（カードなし判定の面積比）
  - RATIO_SINGLE（1枚判定の面積比）
  - WHITE_NEIGHBOR_RADIUS（白隣接チェック半径）

# 4. BLEデバイス要件（ESP32 / M5Atom）

## 4.1 目的
WebアプリからのUNO通知を受信し、LED点灯およびサーボ動作を行う。

---

## 4.2 ハードウェア要件

使用デバイス例

- M5Atom Lite
- AtomS3 Lite
- ESP32開発ボード

接続機器

- LED
- サーボモーター（例：SG90）
- 外部電源

---

## 4.3 BLE機能

- BLE Peripheralとして動作
- 固定デバイス名でアドバタイズ
- サービスUUIDを公開
- 書き込み可能Characteristicを1つ持つ

---

## 4.4 LED制御

UNO通知受信時

- LEDを点灯
- 点灯時間は固定
- 自動消灯

---

## 4.5 サーボ制御

UNO通知受信時

- サーボを指定角度へ回転
- UNO旗を展開する

角度例

    0°  = 旗収納
    60° = 旗展開

サーボ制御方式

- PWM制御

---

## 4.6 電源要件

サーボは外部電源を使用

例

- 単3電池 × 4
- USB 5V
- 5V ACアダプター

注意

- ESP32とサーボ電源のGNDは共有する

---

# 5. インターフェース

## 5.1 Webアプリ → BLEデバイス（スーパーUNOアピールモードのみ）

通信方式

- Bluetooth Low Energy

通信内容

    UNO発生通知

送信データ

    0x01

送信タイミング

    手札状態: multiple → single

# 6. 画面要件

表示項目

- 通常モード選択ボタン
- スーパーUNOアピールモード選択ボタン

---

## 6.2 メイン画面（共通）

表示項目

- カメラプレビュー
- 手札ガイド枠
- 現在の手札状態（none / single / multiple）

消去条件

- 画面をタップ

## 6.4 デバッグ画面（/debug）

表示項目

- カメラプレビュー

- デバッグオーバーレイ（常時表示）
- 手札状態
- 閾値調整スライダー

# 7. 利用手順

1. Webアプリへアクセス
2. モードを選択（通常 / スーパーUNOアピール）
3. カメラ使用許可
4. スーパーモードの場合はBLE接続
5. スマートフォンを首掛け装着
6. 手札をガイド枠内に保持
7. UNO判定時に自動通知・演出
8. 画面をタップして表示を消去

---

# 8. 制約事項

- Chrome for Android前提
- HTTPS環境必須
- カードの重なり方により認識精度が変動
- 照明条件に影響される
- 閾値（SAT_THRESHOLD等）は実行環境に依存するため `/debug` 画面でチューニングが必要

---

# 9. 拡張候補

- LED点滅演出
- サーボ複数制御
- 相手手札監視
- UNO宣言速度計測
- ゲームログ記録

---

# 10. 受け入れ基準

以下を満たすこと

1. Webアプリでインカメラ映像が表示される
2. 手札状態（none / single / multiple）を正しく判定できる
3. 1枚になった際にUNO表示と音声再生が発火する
4. 通常モード・スーパーモードで演出が異なる
5. 画面タップでUNO表示が消える
6. `/debug` 画面で閾値をリアルタイム調整できる
7. スーパーモード時にBLEでESP32へ通知できる
8. ESP32がLED点灯およびサーボ動作を行う
