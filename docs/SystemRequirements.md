# UNO自動宣言装置 要件定義書（試作版）

## 1. 文書概要

### 1.1 目的
UNOプレイ中に、プレイヤーの手札が残り1枚になったことをスマートフォンのインカメラで検知し、Webアプリ上で「UNO」と表示するとともに、BLE接続された外部デバイスへ通知する。  
外部デバイスは通知を受けるとLEDを点灯し、サーボモーターを動作させて「UNO旗」を跳ね上げる。

### 1.2 対象範囲
本要件定義書は以下の2要素を対象とする。

- スマートフォン上で動作するWebアプリケーション  
- BLE接続されるESP32 / M5Atom系デバイス（LED点灯およびサーボ動作）

### 1.3 試作範囲外
本試作では以下は対象外とする。

- UNOカードの種類（色・数字・アクションカード）の識別  
- 音声によるUNO宣言  
- 対戦相手の手札監視  
- UNO公式ルールとの厳密な一致  
- iOS Safariなど全ブラウザ対応  
- インターネット経由の遠隔通信  
- プレイ履歴保存  

---

# 2. システム概要

## 2.1 システム構成

    プレイヤー手札
          │
          ▼
    スマートフォン（インカメラ）
          │
          ▼
    Webアプリ（画像処理）
          │
          ├─ UNO表示
          │
          ▼
        BLE通信
          │
          ▼
    ESP32 / M5Atom
          │
          ├─ LED点灯
          └─ サーボ動作 → UNO旗展開

## 2.2 動作フロー

1. プレイヤーがスマートフォンを首から下げ、インカメラで手札を映す  
2. Webアプリが手札枚数をリアルタイム認識  
3. 手札が「2枚以上 → 1枚」に変化したことを検知  
4. Web画面に「UNO」と表示  
5. BLEでESP32へ通知  
6. ESP32がLEDを点灯  
7. サーボモーターを動かし、UNO旗を跳ね上げる

# 3. Webアプリ要件

## 3.1 動作環境

### 3.1.1 対応端末
- スマートフォン

### 3.1.2 対応ブラウザ
- Google Chrome for Android

### 3.1.3 配信方式
- HTTPSで配信される静的Webアプリ

---

## 3.2 機能要件

### 3.2.1 カメラ取得
- インカメラの映像を取得できること  
- カメラ使用許可をユーザーに要求すること  
- カメラ映像を画面上にプレビュー表示すること  

### 3.2.2 手札ガイド表示
- 手札を映す範囲を画面上に枠で表示する  
- 画像解析はこの枠内を対象とする  

### 3.2.3 手札枚数認識
- カメラ映像からカード枚数を推定できること  
- UNOカードの種類識別は不要  
- 画像処理は端末内で実行する  

想定技術例

- OpenCV  
- OpenCV.js  
- TensorFlow.js（任意）

### 3.2.4 UNO判定
- 直前の枚数と現在の枚数を比較する  
- 「2枚以上 → 1枚」に変化した場合にUNOと判定する  
- 同一状態での連続発火を防止する  

### 3.2.5 画面表示
- 現在の推定手札枚数を表示  
- UNO発生時に画面中央へ大きく「UNO」と表示  
- UNO表示は数秒間継続  

### 3.2.6 BLE接続
- BLEデバイスを検索可能  
- ユーザー操作で接続開始  
- 接続状態を表示  

### 3.2.7 BLE通知送信
- UNO判定時にESP32へデータ送信  
- 送信データは簡易な固定値とする  

送信例

    0x01

または

    "UNO"

### 3.2.8 エラー処理
- カメラ未許可時に警告表示  
- BLE接続失敗時に通知  
- 認識失敗時でもアプリが停止しない

# 4. BLEデバイス要件（ESP32 / M5Atom）

## 4.1 目的
WebアプリからのUNO通知を受信し、LED点灯およびサーボ動作を行う。

---

## 4.2 ハードウェア要件

使用デバイス例

- M5Atom Lite  
- AtomS3 Lite  
- ESP32開発ボード  

接続機器

- LED  
- サーボモーター（例：SG90）  
- 外部電源  

---

## 4.3 BLE機能

- BLE Peripheralとして動作  
- 固定デバイス名でアドバタイズ  
- サービスUUIDを公開  
- 書き込み可能Characteristicを1つ持つ  

---

## 4.4 LED制御

UNO通知受信時

- LEDを点灯  
- 点灯時間は固定  
- 自動消灯  

---

## 4.5 サーボ制御

UNO通知受信時

- サーボを指定角度へ回転  
- UNO旗を展開する  

角度例

    0°  = 旗収納
    60° = 旗展開

サーボ制御方式

- PWM制御  

---

## 4.6 電源要件

サーボは外部電源を使用

例

- 単3電池 × 4  
- USB 5V  
- 5V ACアダプター  

注意

- ESP32とサーボ電源のGNDは共有する  

---

# 5. インターフェース

## 5.1 Webアプリ → BLEデバイス

通信方式

- Bluetooth Low Energy

通信内容

    UNO発生通知

送信タイミング

    手札枚数
    2枚以上 → 1枚

# 7. 利用手順

1. Webアプリへアクセス  
2. カメラ使用許可  
3. BLE接続  
4. スマートフォンを首掛け装着  
5. 手札をガイド枠内に保持  
6. UNO判定時に自動通知

# 8. 制約事項

- Chrome for Android前提  
- HTTPS環境必須  
- カードの重なり方により認識精度が変動  
- 照明条件に影響される

# 10. 受け入れ基準

以下を満たすこと

1. Webアプリでインカメラ映像が表示される  
2. 手札枚数を概算で判定できる  
3. 1枚になった際にUNO表示される  
4. BLEでESP32へ通知できる  
5. ESP32がLED点灯およびサーボ動作を行う
